# Frontend — MarketCortex

React + Vite + TypeScript dashboard for the backend API.

## Setup

    cd frontend
    npm install

## Run (backend must be running on :8000)

    npm run dev      # http://localhost:5173

Set `VITE_API_BASE` (see `.env.example`) if the backend isn't at `http://localhost:8000/api`.

## Build / test / typecheck

    npm run build    # tsc -b && vite build (type-check + bundle)
    npx vitest run   # unit tests

## Pages

- **Dashboard** — pick a ticker and a chart timeframe (1M–5Y) and view the candlestick chart with SMA overlays. Manage your watchlist from a compact, searchable **Watchlist (N) ▾** dropdown (★ toggle on the loaded ticker, filter-and-pick, × to remove). Your **last analysis restores automatically** on load — the reasoning, the Signals list, and the chart's buy/sell markers reappear, tagged *as of* its call date (a pure read; no tokens, no evaluation impact), so you can review past calls without re-running. A **Signals strip** loads with the ticker: the instant **no-LLM opportunity score** (score / reason chips / 🔗 network — same as the Discover board) plus the latest recorded call from each source — **TECH / NET / FAST / DEEP** — with an agree/conflict badge and a 👑 on the per-ticker historically-best source. Click **Analyze with LLM** (scoped to the selected timeframe) to draw buy/sell markers, list the signals (click one for its reasoning), and show the reasoning + key factors + news — or **Deep Analysis** to stream an agentic ReAct run live (collapsible Thought → Action → Observation trace panel); its result renders through the same chart/reasoning path.
- **Portfolio** — a focused opportunity board that scores just your own universe: your **watchlist plus the ticker nodes in your active ontology** (non-ticker `ext:`/`man:` nodes excluded). Because the set is small, **Rescan portfolio** completes in seconds rather than minutes; the button also chains the technical/network snapshot into the Evaluation scoreboard. The board reuses the shared `ScoreBoard` UI — score, call, and reason chips per row; click any row to open the full Dashboard deep-dive. An as-of timestamp, a **Stop** button while scanning, and an empty-state prompt ("add to your watchlist or activate an ontology") complete the page. Beyond its own tab, the portfolio board is the **primary scoring source**: it is overlaid on the broad Discover board and the combined result feeds the Dashboard Signals strip score, the LLM prompts' network section, and Graph node colours/scores — falling back to the broad Discover scan for any ticker not in the portfolio.
- **Discover** — an opportunity board ranking the S&P 500 (or a sector) by a 0–100 no-LLM score; columns include **Exchange** (NASDAQ / NYSE / etc.) and an **S&P** badge (**S&P 500** vs **Custom**); a **search box** above the board filters rows by ticker or company name (client-side). **Custom (non-S&P 500) companies** can be added via the "Add company" form — enter a ticker and the app auto-fetches its name, exchange, sector, and current price, saves it permanently, scans it on the board flagged **Custom**, and lets you remove it with an ×. Filter by sector/direction, choose how many rows to show, **Rescan** with live progress streamed over SSE (a `scanned/total` counter naming the in-flight ticker, plus a **Stop** button — same on the Evaluation action bar and the masthead chip), **Update S&P 500 list**, and click a row to deep-dive on the Dashboard. Every button on the page has a tooltip.
- **Graph** — build and manage **named, versioned ontologies** that drive all analysis. Start by **adding a company** (the empty-canvas button, the **Add company…** sidebar button, or a right-click on the canvas — ticker + optional name), click **Expand neighbours** per node for one-hop live news extraction, **right-click** a node to add a relationship, **rename it** (re-identifying a node — e.g. an imported `ext:` entity — as a Discover-board ticker picks up that ticker's BUY/SELL/HOLD colour and score, rewriting its edges; a name-only change just relabels), delete a node/edge, or toggle **☆ / ★ watchlist** membership. A **find box** at the right end of the ontology toolbar matches tickers/company names and pans + zooms to the picked node. Save the canvas under a user-chosen name via the toolbar (name field + **Save / Save as / New**; up to 5 versions kept). The **Ontologies** sidebar tab lists every saved ontology with an **ACTIVE** badge on the live one, **Set active** per row, a "None (network signal off)" row, and version history (load an old revision to inspect it; the canvas marks dirty until you save). **Exactly one ontology is active** — it is the only graph all scoring paths consume (Discover NET blend, Dashboard score chip, LLM prompts' network section, portfolio snapshots); no active ontology means no network signal anywhere. Activating, saving over, or deleting any version of the active ontology immediately re-bakes the Discover snapshot. A hint on the canvas shows which ontology analysis is currently using (or "no network signal") when the canvas isn't the live active revision. The **Import** sub-tab accepts a small app-defined JSON model (paste/upload; built-in **Copy ChatGPT prompt** template) as a reusable building block — merge it into the canvas via a conflict-resolution MergePreview (links imported companies to the Discover list, collapses clashing nodes, dedupes relationships), then save to make it live. Imported entities render distinctly (violet external nodes, dashed edges). Lazy-loaded so the graph library stays out of the main bundle; a collapsible **legend** on the canvas explains node/edge colours and styles.
- **Evaluation** — tracks how accurate past calls turned out to be across **all four signal sources** (Technical / Network / LLM fast / LLM deep): per-source scoreboard cards (calls / scored / hit-rate / grade) and a per-company board (hit-rate, 0–100 score, Strong/Mixed/Weak grade, overconfidence flag) that expands to each call's 1d/5d/20d ✓/✗ outcomes with a source badge per row and a source filter for the call list, plus an on-demand source-aware **Explain miss** LLM post-mortem, a confirm-guarded **Clear all results** reset (start the experiment over after a period of testing), plus an **action bar** that runs processes across your **portfolio (watchlist + active ontology)** — buttons are **Rescan portfolio** (re-scores the portfolio and chains the technical/network snapshot), **Fast LLM analysis**, and **Deep LLM analysis** (live per-ticker chips + Stop; already-recorded tickers are skipped; all processes live at app level so they survive page navigation, with a pulsing masthead chip showing progress from any page; the old standalone snapshot button and the full-Discover-rescan button are gone — Rescan portfolio covers both) — with a live **when-to-run hint** showing whether the US market is open and the next close in the user's local timezone (run after the close so daily prices are final; the same hint sits on the Discover bar, and the Dashboard shows it only while the market is open).
- **Settings** — choose the LLM provider (Anthropic/OpenAI/Gemini/DeepSeek/Ollama), set model + API key (or Ollama base URL), test the connection, edit the watchlist, and toggle the Truth Social / screener / network signals.

## Notes

- Built on the Vite 5 toolchain (vitest 2, lightweight-charts 4, react-force-graph-2d 1) for Node 20 compatibility.
- The backend API uses the `period` query param. `dividend_yield` from yfinance is already a percentage value (e.g. 0.35 = 0.35%), shown as-is.
